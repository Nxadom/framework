Kenapa NexaUI Mudah?

Ringkas, intuitif, tanpa boilerplate berlebihan.

route

Routing Minimal

$router->add('/', 'FrontendController@index') — satu baris, selesai. Frontend routing otomatis untuk /product, /mall tanpa daftar manual.

mediation

Controller 3 Langkah

assignVars() → divert() → render(). Tanpa konfigurasi kompleks, langsung ke template.

storage

Database Tanpa Model

$this->Storage('users')->get(). CRUD, join, paginate — tanpa definisi model jika tak diperlukan.

code

Template Sederhana

{variable}, {price|rp}, {if} — syntax jelas, filter built-in.

link

Reuse Tanpa DI

refParams(), useModels(), useParams() — panggil controller/model lain tanpa dependency injection.

devices

Multi-Device & API

divert() pilih layout theme/mobile/tablet. REST API via routes/api.php — siap pakai.

View Hanya HTML

Pemisahan jelas antara Frontend dan Backend — poin menarik NexaUI.

vertical_split

Desainer ↔ Developer

Kode tampilan ditulis murni di file .html. Tidak ada PHP di template — hanya variable {var}, block, dan filter.

Desainer UI fokus di struktur HTML & CSS. Developer backend fokus di Controller & Model. Kolaborasi bersih tanpa tabrakan.

templates/theme/product/index.html
<div class="product-list">
<!-- NEXA row -->
    <div class="item">
        <h3>{title}</h3>
        <p>{price|rp}</p>
    </div>
<!-- END row -->
</div>

Murni HTML + placeholder. Data di-inject dari Controller via assignVars() & nexaBlock().
